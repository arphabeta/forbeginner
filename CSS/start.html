CSS (Cascading Style Sheets)

*CSS 개발환경 세팅하는 방법

    CSS는 웹을 꾸밀 때 사용하는 언어다.

    CSS는 <head>태그 속 <style>태그에 css를 적용한다. HTML에서 했던 것처럼 태그 내에 직접 폰트 색이나 문단 배열을 설정할 수 있지만, 간단한 웹을 만들 때도 수식이 길어지기 때문에 되도록이면 <style>tag를 사용하거나, CSS 페이지를 따로 만든 후 연결하는 것이 좋다.

  1. <style>태그를 사용하는 방법

    2. <style>태그를 사용하는 CSS의 기본 세팅은 아래와 같이 할 수 있다. 반드시 <head>안에 <style>태그를 작성해야 적용된다. <body>태그 안에 적힌 HTML 문법들을 꾸며주는 역할을 하기 때문에 꾸며주는 코드는 노출이 되면 곤란하므로 <head>부분에 넣는 것이다.

    HTML기본세팅 <!DOCTYPE html>
    <html>
    <head>

    <meta charset="utf-8">

    <title>css</title>

    <style>

    </style>

    </head>

    <body>

    </body>

    </html>

HTML 문서 세팅을 할 때 charset 은 <meta charset="utf-8">로 작성했지만, CSS에서는 @charset "utf-8"; 로 작성해야 한다. 나도 아직은 잘 모르지만 컴퓨터 언어마다 코드를 하는 방식이 다르기 때문에 이런 점을 고려해서 모두 작성해야 한다고 한다. 나는 이걸 한국어는 문장을 마칠 때.(작은 점)을 쓰지만 일본어로는 。(크고 동그란 점)을 써야 하는 것과 비슷하다고 생각했다. 언어가 다르듯 글을 시작하고 마치는 방법, 비슷한 언어라도 단어(코드)를 읽고, 쓰는 방법이 다르다고 생각했다.

      4. html 프로젝트 파일에 css 프로젝트 파일을 연결해주려면 <head>태그 안에 <link>를 사용하여 css 파일을 삽입해줘야 한다. 1. 번의 <style>태그를 <head>태그 안에 작성한 것과 동일한 이유이다. 연결하지 않은 채로 내버려 두면 서로 독립적인 파일로 존재하게 된다. 그럼 (하고 싶은 파일 이름).css 파일에 아무리 정성스럽고 길게 코드를 입력한다고 해도 css는 적용하려고 만든 html 문서에 적용되지 않는다.
